# 포챔 AI M4 검증 프로토콜

## 판정 원칙

자동 테스트 통과는 구현 안정성 판정이며, 플레이 성능 향상 판정이 아니다. 아래 세 집합을 서로 분리해 기록한다.

1. 화면 인식 오류: 게임 화면의 사실을 잘못 구조화함
2. 상태 확정 오류: 사용자가 잘못된 값을 확정함
3. 전략 오류: 확정 상태는 맞지만 추천의 사후 가치가 낮음

## 자동 검증

- `npm run data:sync`: 날짜 고정 M-B 스냅샷을 다시 만들고 생성 코드와 일치시킨다.
- `npm run typecheck && npm test && npm run build`: 스키마, 팀 검증, 60개 미리보기 후보, 기술/교체 비교, NIM 파서를 검사한다.
- `npm run benchmark`: M-B 전 종·폼 참조가 끊기지 않았는지 확인하고 1,000개 추천 상태의 p50/p95 시간을 기록한다. p95 2초 이상 또는 미지원 폼 1개 이상이면 실패한다.

이 벤치마크의 `meanSimulationEstimate`는 모델 자체의 평균 출력값일 뿐 실제 승률이 아니다.

## 한국어 BlueStacks 골든 프레임 100장

팀 미리보기 25장, 기술 선택 25장, 교체/기절 전환 20장, 날씨·필드·상태가 있는 턴 20장, 결과 10장을 수집한다. 해상도·UI 배율·밝기 조건을 함께 기록하고 동일 장면의 중복은 제외한다.

각 프레임의 종/폼, 페이즈, 활성 포켓몬, HP, 상태, 보이는 기술을 사람이 정답 라벨링한다. 핵심 필드 정확도 95% 이상이며, 읽을 수 없는 값은 오답 대신 `unknown`이어야 합격한다. 캡처부터 확인 초안까지 p95 8초 이하를 별도로 계산한다.

## 1,000개 정책 페어링

동일 팀·상대 가설·난수 시드마다 탐색 정책과 즉시 최대 데미지 정책을 각각 완결 경기로 실행한다. 검색 정책의 승률 차이가 5%p 이상이고 부트스트랩 95% 신뢰구간 하한이 0보다 커야 합격한다. 현재 근사 엔진만으로 완결 경기를 재현하지 못하면 이 항목은 `미검증`으로 남기며 대체 지표로 통과 처리하지 않는다.

## 친선 30판

- 같은 상대와 팀 일정을 30판 준비하고 도움 없음 15판, AI 도움 15판의 순서를 무작위 배정한다.
- 매 턴 인식 수정값, 추천 3개, 채택/거부, 실제 행동, 결과, NIM/로컬 처리 시간을 기록한다.
- 경기 후 전체 공개 정보를 반영해 사후 최선 행동을 다시 계산하고 의사결정 후회를 기록한다.
- AI 구간의 평균 후회가 20% 이상 감소하고 승수가 무도움 구간보다 낮지 않을 때만 성능 향상을 주장한다.

## 필수 로그 필드

`matchId`, `condition`, `turn`, `captureHash`, `visionConfidence`, `correctedFields`, `confirmedState`, `recommendations`, `acceptedAction`, `actualAction`, `nimLatencyMs`, `engineLatencyMs`, `result`, `postHocBestAction`, `decisionRegret`를 보존한다. 캡처 원본은 기본 보존하지 않으며 골든 프레임으로 명시 동의한 자료만 별도 관리한다.
